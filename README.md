---

## 🧠 Project idea

The goal is not just to write code, but to **separate responsibilities clearly**, like in real-world applications.

I follow this structure:

* 🟦 **Data** → input from the user or stored information
* 🟩 **Logic** → functions that process the data
* 🟧 **Output** → what is shown to the user (print statements)

---

## 🏗️ Structure example

```
my_app/
├── main.py        # controls the flow of the program
├── utils.py       # logic functions
├── main2.py       # practice mini-projects
```

---

## ⚙️ How it works

1. The program gets **input (data)** from the user
2. The data is processed using **functions (logic)**
3. The result is displayed using **print (output)**

---

## 🚀 Why this matters

Separating data, logic, and output helps to:

* Keep code clean and organized
* Make it easier to debug
* Allow reuse of functions in other projects
